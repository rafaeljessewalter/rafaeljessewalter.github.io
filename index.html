Project Brief — DooPlay-style Static Website (for Vercel) Goal: Build a complete static website (NOT a mobile/desktop application) that visually and functionally clones the demo site https://demo.ews.pe/dooplay/. The site must be production-ready for hosting on Vercel and must read movie/TV IDs from a runtime JSON file (no database required). High-level requirements Static site: HTML/CSS/JS only (or a static framework like Next.js exported as static), deployable to Vercel as a static site. If serverless helpers are used they must be minimal and optional. Pages to include (match the demo behavior & navigation): Home / Featured (index.html) Movies listing (movies.html) TV listing (tv.html) Popular (popular.html) Top (top.html) Genres (genres.html) Year filter (year.html) Search (search.html) Movie detail (movie-detail.html) TV detail (tv-detail.html) Watch / player (watch.html) 404 page JSON-driven content: All lists must be populated from a single JSON file in the repository (example: /data/content.json) that contains items with at minimum: type: "movie" or "tv" id: TMDb numeric ID optional: year, link (player url or m3u8), poster_path (override), title (override) TMDb integration: When a TMDb key is provided, the client should fetch details (title, poster, overview, rating, backdrop) using the TMDb API and enrich entries at runtime. Provide two secure options in the deliverable and implementation: Option A (simple): Client-side TMDb key set in a public file (assets/js/config.js) — clearly document security implications. Option B (recommended): Optional Vercel serverless function /api/tmdb?type=movie&id=... that proxies requests to TMDb using a Vercel environment variable TMDB_API_KEY, keeping the API key secret.
